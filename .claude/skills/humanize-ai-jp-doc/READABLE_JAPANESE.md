# 読みやすい日本語の定義

読みやすい日本語 = **わかりやすい** × **読みやすい**。

- **わかりやすい** — 一読で意味が取れる。主語・述語が明確。誰にでもわかる言葉を使う
- **読みやすい** — リズムがある。文末にバリエーションがある。流れが途切れない

---

## 結論から書く

各セクション・各段落の1文目に結論を置く。手順・根拠は2文目以降。

| 悪い例 | 良い例 |
|---|---|
| 「登録ボタンを押すと入力チェックが動く。〜定義と一致している」 | 「登録処理は仕様書通りに動作している。〜」 |
| 「更新時は存在確認後にSQLを実行する。〜一致している」 | 「更新・削除処理はどちらも仕様書と一致している。〜」 |

---

## 人間らしさの正体

人間の文章は状況を限定する。AIの文章は汎用化してぼかす。

「いつ」「何を」「何に対して」を具体的に書く。

| 悪い例 | 良い例 |
|---|---|
| 定期的に見直しを推奨しています | バージョン更新後は設定を確認してください |
| 業務効率化を実現します | 作業時間が半分になります |
| 品質向上に寄与します | バグ検出率が上がります |
| 適切にご利用いただけます | （削除。具体的に何ができるかを書く） |

---

## 見出し

| レベル | 用途 |
|---|---|
| `#` | ドキュメントのタイトル（1つだけ） |
| `##` | 大きなトピックの区切り |
| `###` | `##` の中のサブトピック |

- `####` 以下は使わない
- 見出し直後に本文を入れる。見出しを連続させない
- 見出しは体言止め

---

## 箇条書きと文章

**箇条書きにする:** 操作手順（番号付き）、3つ以上の並列項目

**文章にする:** 説明・理由・背景、2つ以下の列挙、因果関係、注意・補足

---

## インデント

- 箇条書きの補足説明、手順内の注意書き → インデントする
- 本文段落、見出し直下、独立した補足 → インデントしない
- 深さは1段まで

---

## 改行

| 場所 | ルール |
|---|---|
| 文の途中（本文） | 40字に達したらその位置で折る。読点の位置は関係ない |
| 箇条書き項目内 | 改行しない。読点を含む場合は40字以内に収まるよう文言を短く書き直す |
| 段落と段落の間 | 空行1行を入れる |
| 見出しの直前 | 空行1行を入れる |
| 見出しの直後 | 空行なし。本文をすぐ書く |
| 箇条書きの項目間 | 空行なし |
| 箇条書きブロックの前後 | 空行1行を入れる |
| 文末の読点 | 使わない。文末は句点「。」で終える |

---

## 変換ルール

| Before | After |
|---|---|
| 〜することができます | 〜できます |
| 〜を行います | 〜します |
| 〜が提供されています | 〜を使えます |
| こそあど言葉（これ・その・ここ） | 指示対象の名詞に置き換える |
| 本ドキュメントでは〜説明します | （削除） |
| また・さらに・なおの連発 | 一つだけ残すか削除 |
| 一文に情報が複数 | 一文一情報に分割 |
| 一行が40字に達した | 読点・文末・語の境界を問わず40字目で折る |
| 修飾のネストが深い | 修飾語を外に出して別文にする |
| 抽象名詞で終わる（効率化・品質向上） | 「何が」「どう変わるか」を続けて書く |
| ご不明な点があれば〜 | （削除） |
| 不要な横文字・専門用語 | 日本語に置き換える |
| 言葉を削って意味が消えた | 削らず短い言葉で言い直す |
