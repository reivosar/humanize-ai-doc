# 読みやすい日本語の定義

## 前提

良い文章とは「読む文章」ではなく「見てわかる文章」だ。

人の目は文字を読む前に、ページの形を見る。
行の長さ、空白の位置、段落の切れ目——これらが意味の単位を視覚的に伝える。
だから、短い文、空行、1文1行はスタイルの好みではない。
「見ただけで構造がわかる」ための装置だ。

この前提からすべてのルールが導かれる。

---

## 原則

平易な言葉で書く。
読み手が途中で止まらず、最後まで読み進められる。

- 1文1行で書く。縦に流し、目で追えるようにする
- 主語を省略しない
- 受け身を使わない（「〜される」→「〜する」）
- 用語を説明なしに使わない
- 一文に情報は一つ
- 抽象語で終わらない（「効率化」→「何が何分短くなるか」）
- 長くなる文は箇条書きにする

---

## 結論から書く

各セクション・各段落の1文目に結論を置く。手順・根拠は2文目以降。
一つのパラグラフに書く話題は一つだけ。関係のない文を混ぜない。
情報が複数ある場合、冒頭で件数を示す（例：「理由は3つある」）。

---

## 人間らしさの正体

人間の文章は状況を限定する。AIの文章は汎用化してぼかす。
「いつ」「何を」「何に対して」を具体的に書く。

| 悪い例 | 良い例 |
|---|---|
| 業務効率化を実現します | 作業時間が半分になります |
| 定期的に見直しを推奨しています | バージョン更新後は設定を確認してください |

---

## 見出し

| レベル | 用途 |
|---|---|
| `#` | ドキュメントのタイトル（1つだけ） |
| `##` | 大きなトピックの区切り |
| `###` | `##` の中のサブトピック |

- `####` 以下は使わない
- 見出し直後に本文を入れる。見出しを連続させない
- 見出しは原則として体言止め。流れが不自然になる場合は動詞止めも可

---

## 箇条書きと文章

**箇条書きにする:** 操作手順（番号付き）、3つ以上の並列項目、一文に収まらない列挙

**文章にする:** 説明・理由・背景、2つ以下の列挙、因果関係、注意・補足

---

## インデント

- 箇条書きの補足説明、手順内の注意書き → インデントする
- 本文段落、見出し直下、独立した補足 → インデントしない
- 深さは1段まで

---

## 改行

| 場所 | ルール |
|---|---|
| 本文の各文 | 1文1行で書く。文の途中で折らない |
| 意味の切れ目 | 空行1行を入れる |
| 関連する短文のまとまり | 空行なしで並べてもよい |
| 見出しの直前 | 空行1行を入れる |
| 見出しの直後 | 空行なし。本文をすぐ書く |
| 箇条書きの項目間 | 空行なし |
| 箇条書きブロックの前後 | 空行1行を入れる |
| 文末の読点 | 使わない。文末は句点「。」で終える |

---

## 変換ルール

| Before | After |
|---|---|
| 〜することができます | 〜できます |
| 〜を行います | 〜します |
| 〜が提供されています | 〜を使えます |
| こそあど言葉（これ・その・ここ） | 指示対象の名詞に置き換える。前文だけで参照先がわかること |
| また・さらに・なお・加えて | 削除（だから・しかし・一方で のみ可） |
| 否定形（〜しないと〜できない） | 肯定形（〜すると〜できる） |
| 比較対象なし（「速度が向上した」） | 数値で両側を示す（「10秒から3秒になった」） |
| 曖昧な量（多くの・すぐ） | 数字で書く（120人・3分） |
| 修飾語が長く主語と述語が離れている | 主語を先に出し、修飾語を別文にする |
| 不要な横文字・専門用語 | 日本語に置き換える。どうしても使う場合は直後に括弧で説明する |
| 本ドキュメントでは〜説明します | （削除） |
| ご不明な点があれば〜 | （削除） |
| 一文が長い | 読点の位置で分割するか、箇条書きにする |
| 修飾のネストが深い | 修飾語を外に出して別文にする |
| 抽象名詞で終わる（効率化・品質向上） | 「何が」「どう変わるか」を続けて書く |
| 言葉を削って意味が消えた | 削らず短い言葉で言い直す |

---

## 重複禁止

同じ内容を繰り返さない。
新しい文は、直前の文にない情報を最低1つ含む。

AIの文章は同じ主張を言い換えて3回繰り返す。
人間の文章は「わかったなら次に進む」。
